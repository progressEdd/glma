# Plan 03-02: Jupyter Notebook Parsing, Variable Tracking, Notebook Compaction

## Objective

Build Jupyter notebook (.ipynb) compaction: parse notebooks with nbformat, extract per-statement variable definitions and references, track cross-cell variable flow, and generate compacted markdown output. This is a parallel track to Plan 03-01 — both are in Wave 1 with no dependencies on each other.

## Why Wave 1 (Parallel)

Notebook compaction is independent of the source file query formatter. It uses nbformat + Python's `ast` module, not the LadybugStore. The CLI integration (detecting `.ipynb` extension and dispatching) is a thin layer added in Plan 03-03.

**Data classes** (use `dataclasses` from stdlib):

```python
@dataclass
class StatementInfo:
    """Variable info for a single statement within a cell."""
    line_number: int          # 1-indexed line within the cell
    statement_type: str       # "assign", "function_def", "class_def", "import", "for_loop", "with_stmt", "aug_assign"
    defines: list[str]        # Names defined by this statement
    references: list[str]     # Names referenced by this statement

@dataclass
class CellVariableInfo:
    """Variable info for a single notebook cell."""
    cell_index: int           # 0-indexed cell number
    cell_type: str            # "code" or "markdown"
    source: str               # Raw cell source
    statements: list[StatementInfo]  # Per-statement analysis (empty for markdown cells)
    all_defines: list[str]    # Aggregated: all names defined in this cell
    all_references: list[str] # Aggregated: all names referenced in this cell
```

**Function: `extract_cell_variables(cell_source: str, cell_index: int) -> CellVariableInfo`**

1. Try: `tree = ast.parse(cell_source)`
2. If SyntaxError: return `CellVariableInfo(cell_index=cell_index, cell_type="code", source=cell_source, statements=[], all_defines=[], all_references=[])` — graceful degradation
3. Walk `tree.body` (list of top-level statements) — NOT `ast.walk`, because we need per-statement granularity
4. For each statement node, determine statement_type and extract defines/references:

   - `ast.Assign`: defines = [target.id for each Name in node.targets], references = all Name(ctx=Load) in node.value
   - `ast.AugAssign`: defines = [node.target.id], references = all Name(ctx=Load) in node.value + [node.target.id] (aug assign reads the variable too)
   - `ast.FunctionDef`: defines = [node.name], references = all arg names + decorators + default values
   - `ast.AsyncFunctionDef`: same as FunctionDef
   - `ast.ClassDef`: defines = [node.name], references = all base classes + decorators
   - `ast.Import`: defines = [alias.asname or alias.name.split('.')[0] for each alias], references = []
   - `ast.ImportFrom`: defines = [alias.asname or alias.name for each alias], references = []
   - `ast.For` / `ast.AsyncFor`: defines = names in node.target (could be Name or Tuple), references = all Name(ctx=Load) in node.iter
   - `ast.With` / `ast.AsyncWith`: defines = [item.optional_vars.id for each item if optional_vars], references = [item.context_expr names]
   - `ast.Return`: defines = [], references = all Name(ctx=Load) in node.value
   - `ast.Expr`: defines = [], references = all Name(ctx=Load) in node.value
   - All other statement types: defines = [], references = [] (don't crash on unexpected types)

5. For each statement, create a `StatementInfo` with line_number = `node.lineno` (1-indexed)

6. Helper function `_extract_names(node) -> list[str]` that recursively extracts all `ast.Name` IDs with `ctx=Load` from any AST node. Use a recursive walk (not ast.walk, which visits all descendants indiscriminately — we want to collect names but skip nested function/class definitions to avoid counting inner-scope names):

```python
def _extract_name_refs(node) -> list[str]:
    """Extract all Name references (Load context) from an AST subtree."""
    names = []
    for child in ast.iter_child_nodes(node):
        if isinstance(child, ast.Name) and isinstance(child.ctx, ast.Load):
            names.append(child.id)
        elif isinstance(child, (ast.FunctionDef, ast.AsyncFunctionDef, ast.ClassDef)):
            continue  # Don't descend into nested scopes
        else:
            names.extend(_extract_name_refs(child))
    return names
```

7. Helper function `_extract_definition_names(node) -> list[str]`:
   - `ast.Name` with `ctx=Store` → return [node.id]
   - `ast.Tuple` / `ast.List` → recurse on each element
   - Otherwise → return []

8. Aggregate `all_defines` and `all_references` from the statements list (deduplicate with `list(dict.fromkeys(...))` to preserve order)

9. Return `CellVariableInfo`

**Function: `build_variable_flow(cells: list[CellVariableInfo]) -> dict[str, dict]`**

Builds the cross-cell variable flow map. For each variable defined anywhere:

```python
{
    "model": {
        "defined_in": [{"cell": 1, "line": 3}],  # All definition sites
        "used_in": [{"cell": 3, "line": 1}, {"cell": 5, "line": 2}]  # All usage sites (excluding definition cell)
    }
}
```

Algorithm:
1. Track `defined_vars: dict[str, list[dict]]` — for each variable, list of {cell, line} where defined
2. Track `used_vars: dict[str, list[dict]]` — for each variable, list of {cell, line} where referenced
3. First pass: populate defined_vars from each cell's all_defines (with cell_index and line from statements)
4. Second pass: for each cell's all_references, if the referenced name is in defined_vars, add to used_vars
5. Return the merged dict
</action>
