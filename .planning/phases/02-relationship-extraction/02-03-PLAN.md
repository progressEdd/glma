# Plan 02-03: Pipeline Integration, Confidence Tagging, Relationship Markdown Output, Cross-File Resolution

## Objective

Integrate relationship extraction into the indexing pipeline as a second pass (after all chunks are stored), add confidence tagging logic, update the markdown writer to include both inline per-chunk relationship tables and a summary `## Relationships` section, and handle cross-file relationship resolution where the target chunk is in a different indexed file.

```python
# Pass 2: Extract relationships (requires all chunks in DB for cross-file resolution)
if progress:
    progress._console.print("[dim]Extracting relationships...[/dim]")

source_files_info = []  # Collect (filepath, language) from pass 1

# Re-collect the source files (same list, same filtering)
for filepath, language_name in source_files:
    source_files_info.append((filepath, Language(language_name)))

for filepath, language in source_files_info:
    relative_path = str(filepath.relative_to(repo_root))

    # Skip files that weren't processed (unchanged files still have chunks in DB)
    # Only re-extract relationships for changed/new files
    stored_hash = indexed_files.get(relative_path)
    try:
        current_hash = file_content_hash(filepath)
    except (OSError, IOError):
        continue

    if stored_hash != current_hash:
        # File was changed — re-extracting relationships
        pass  # Continue below
    elif relative_path in source_paths and stored_hash == current_hash:
        # File unchanged but may need relationship refresh if dependencies changed
        # For now, skip unchanged files — their relationships are already in DB
        continue

    # Get chunks for this file from the DB
    chunks_result = store.conn.execute(
        "MATCH (c:Chunk {file_path: $fp}) RETURN c.id, c.name, c.chunk_type, c.file_path, c.content, c.summary, c.start_line, c.end_line, c.content_hash, c.parent_id",
        {"fp": relative_path},
    )
    chunks = []
    for row in chunks_result:
        chunks.append(Chunk(
            id=row[0], name=row[1], chunk_type=ChunkType(row[2]),
            file_path=row[3], content=row[4], summary=row[5] or None,
            start_line=row[6], end_line=row[7], content_hash=row[8],
            parent_id=row[9] or None,
        ))

    if not chunks:
        continue

    # Extract relationships
    relationships = extract_relationships(filepath, language, repo_root, chunks, store)
    store.upsert_relationships(relative_path, relationships)

    # Re-write markdown with relationships
    file_rels = store.get_file_relationships(relative_path)
    write_markdown(chunks, repo_root, config.output_dir, relationships=file_rels)
```

Add to `pipeline.py` after the relationship extraction pass:

1. After all relationships are stored, do a final markdown rewrite pass for cross-file incoming relationships:

```python
# Pass 3: Final markdown with cross-file incoming relationships
for filepath, language in source_files_info:
    relative_path = str(filepath.relative_to(repo_root))

    # Get all relationships involving this file's chunks (both directions)
    all_rels = []

    # Outgoing from this file
    out_rels = store.get_file_relationships(relative_path)
    all_rels.extend(out_rels)

    # Incoming to this file's chunks from other files
    chunks_result = store.conn.execute(
        "MATCH (c:Chunk {file_path: $fp}) RETURN c.id",
        {"fp": relative_path},
    )
    chunk_ids = [row[0] for row in chunks_result]

    for cid in chunk_ids:
        incoming = store.get_incoming_relationships(cid)
        for rel in incoming:
            # Only include cross-file incoming (skip self-referential and same-file)
            source_result = store.conn.execute(
                "MATCH (c:Chunk {id: $sid}) RETURN c.file_path",
                {"sid": rel["source_id"]},
            )
            source_rows = list(source_result)
            if source_rows and source_rows[0][0] != relative_path:
                all_rels.append({
                    "source_id": rel["source_id"],
                    "source_name": rel["source_chunk_name"],
                    "rel_type": rel["rel_type"],
                    "confidence": rel["confidence"],
                    "source_line": rel["source_line"],
                    "target_name": relative_path,  # This file
                    "target_id": cid,
                    "direction": "incoming",
                })

    # Get chunks
    chunks = _load_chunks_from_store(store, relative_path)

    if chunks:
        write_markdown(chunks, repo_root, config.output_dir, relationships=all_rels)
```

2. Add helper `_load_chunks_from_store(store, file_path) -> list[Chunk]` that queries the store for chunks and reconstructs Chunk objects.

3. Update the writer to handle the `direction` field in relationship dicts for proper "Imported By" / "Called By" cross-file display.

**Performance note:** This third pass does one DB query per file for incoming relationships. For repos with 10K files, this is 10K queries. Each query is a simple MATCH on indexed fields (chunk id/file_path), so should be fast. If performance is an issue, batch the queries, but start simple.
</action>
