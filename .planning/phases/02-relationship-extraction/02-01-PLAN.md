# Plan 02-01: Relationship Data Model, Ladybug RELATES_TO Table, C Relationship Extraction

## Objective

Add the relationship data model (RelType, Confidence enums + Relationship Pydantic model), create the RELATES_TO edge table in LadybugStore with CRUD methods, extend parser configs with relationship-relevant C AST node types, and implement C relationship extraction from tree-sitter ASTs (function calls + #include resolution).

Create new file `02-worktrees/glma/src/glma/index/relationships.py` with the following implementation:

```python
"""Relationship extraction from tree-sitter ASTs.

Extracts structural relationships (calls, imports/includes, inheritance) from
parsed ASTs and resolves them against chunks stored in the database.
"""

from pathlib import Path
from typing import Optional

from tree_sitter import Node

from glma.models import Chunk, Language, Relationship, RelType, Confidence
from glma.index.parser import PARSER_CONFIGS, get_root_node
from glma.db.ladybug_store import LadybugStore
```

1. `_extract_c_calls(node: Node, source_chunks: list[Chunk], file_path: str) -> list[Relationship]`:
   - Walk AST looking for `call_expression` nodes
   - For each `call_expression`, the first child is the callee
   - If callee is `identifier` → extract callee name as `target_name`
   - Find matching chunk in `source_chunks` where `chunk.name == callee_name`
   - If found: create Relationship with `rel_type=CALLS`, `confidence=DIRECT`, `target_id=chunk.id`
   - If not found: create Relationship with `rel_type=CALLS`, `confidence=INFERRED`, `target_id=""`, `target_name=callee_name`
   - `source_id` = the chunk ID of the enclosing function_definition (walk parent chain to find it)
   - `source_line` = call_expression start line (1-indexed)

2. `_extract_c_includes(node: Node, source_chunks: list[Chunk], file_path: str, indexed_files: dict[str, str]) -> list[Relationship]`:
   - Walk AST looking for `preproc_include` nodes
   - For `string_literal` child (local include like `"myheader.h"`): extract the include path from the `string_content` child
   - Resolve: search `indexed_files` for a file_path ending with the include name
   - If found: `confidence=DIRECT`, `rel_type=INCLUDES`
   - If not found: `confidence=INFERRED`, `rel_type=INCLUDES`
   - For `system_lib_string` child (system header like `<stdio.h>`): always `confidence=INFERRED`
   - `source_id` = first chunk in the file (or create a module-level reference), `source_line` = preproc_include line

3. `_find_enclosing_chunk(node: Node, chunks: list[Chunk]) -> Optional[Chunk]`:
   - Walk up the node's parent chain to find the nearest `function_definition` or other chunk-type node
   - Match by start_line to find the corresponding Chunk object
   - Return None if no enclosing chunk found (module-level code)

4. `extract_c_relationships(filepath: Path, language: Language, repo_root: Path, source_chunks: list[Chunk], store: LadybugStore) -> list[Relationship]`:
   - Get root node from tree-sitter
   - Get indexed files from store: `store.get_indexed_files()`
   - Call `_extract_c_calls` and `_extract_c_includes`
   - Return combined list of Relationships

5. `extract_relationships(filepath: Path, language: Language, repo_root: Path, source_chunks: list[Chunk], store: LadybugStore) -> list[Relationship]`:
   - Dispatcher function that calls `extract_c_relationships` for C language
   - Returns empty list for unsupported languages (Python handled in Plan 02-02)
</action>
