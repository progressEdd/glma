# Plan 02-02: Python Relationship Extraction, Import Alias Resolution, self.method() Resolution

## Objective

Implement Python relationship extraction from tree-sitter ASTs covering: function calls (direct and attribute-based), import extraction with alias tracking, class inheritance extraction, import alias resolution (including `from X import Y`, `import X; X.y()`, aliased imports), and `self.method()` resolution to class methods. Build a per-file import map resolver that enables cross-file relationship targeting.

## Background

Plan 02-01 established the data model, Ladybug RELATES_TO storage, and C relationship extraction. This plan adds Python-specific extraction â€” the more complex language due to dynamic features like import aliases, `self` references, and attribute-based calls. Python ASTs use `call` nodes (not `call_expression`), `import_statement`/`import_from_statement` for imports, and `class_definition` with `argument_list` for inheritance.

```python
"""Cross-file resolution for Python relationships.

Builds per-file import maps and resolves callee names to target chunks
using the LadybugStore database.
"""

from dataclasses import dataclass
from pathlib import Path
from typing import Optional

from tree_sitter import Node

from glma.models import Language
from glma.db.ladybug_store import LadybugStore
```

1. `@dataclass ImportInfo`:
   - `local_name: str` â€” name used in the importing file (alias or imported name)
   - `source_module: str` â€” dotted module path (e.g., "pathlib", "foo.bar")
   - `imported_name: Optional[str]` â€” specific name imported (e.g., "Path" for `from pathlib import Path`), None for module-level import

2. `build_import_map(node: Node) -> dict[str, ImportInfo]`:
   - Walk the AST root node looking for `import_statement` and `import_from_statement` nodes
   - For `import_statement`:
     - Child `dotted_name` â†’ module path, `identifier` (after `as`) â†’ local name
     - If `aliased_import` child: extract `dotted_name` (source) and last `identifier` (alias)
     - Example: `import os` â†’ `{"os": ImportInfo("os", "os", None)}`
     - Example: `import foo.bar as baz` â†’ `{"baz": ImportInfo("baz", "foo.bar", None)}`
     - Example: `import foo.bar` â†’ `{"foo": ImportInfo("foo", "foo.bar", None)}` (use first component as local_name)
   - For `import_from_statement`:
     - First `dotted_name` after `from` â†’ source module
     - Remaining `dotted_name` children after `import` keyword â†’ imported names
     - Example: `from pathlib import Path` â†’ `{"Path": ImportInfo("Path", "pathlib", "Path")}`
     - Example: `from typing import Optional, List` â†’ two entries: `{"Optional": ImportInfo("Optional", "typing", "Optional"), "List": ImportInfo("List", "typing", "List")}`
   - Return dict mapping local_name â†’ ImportInfo

3. `resolve_callee(callee_node: Node, import_map: dict[str, ImportInfo], enclosing_class: Optional[str], store: LadybugStore, file_path: str) -> tuple[str, str]`:
   - Takes a callee node from a `call` expression and returns `(target_id, target_name)` where target_id is "" if unresolved
   - Cases:
     a. `identifier` callee (e.g., `foo()`): Look up "foo" in import_map. If found, try to resolve via store. If not in import_map, look for same-file chunk with name "foo".
     b. `attribute` callee with object `self` (e.g., `self.method()`): Resolve to method in `enclosing_class`. Query store for chunk with name matching the attribute, `file_path` matching current file, and `parent_id` containing the class name.
     c. `attribute` callee with object matching import_map (e.g., `os.path.join()` or `baz.something()`): Use import_map to resolve object â†’ module. Query store for chunks in the resolved module file with name matching the attribute.
     d. `attribute` callee with unknown object (e.g., `obj.method()`): Return `("", "obj.method")` â€” INFERRED.
     e. Nested `attribute` (e.g., `os.path.join()`): Resolve outermost object, then chain. For this phase, resolve the first level and mark as INFERRED for deeper chains.
   - Store queries use: find chunks where `name` matches and (if applicable) `file_path` matches the resolved module path (convert dots to slashes: "pathlib" â†’ "pathlib.py", "foo.bar" â†’ "foo/bar.py")

4. `find_enclosing_class(node: Node) -> Optional[str]`:
   - Walk up the parent chain from a node looking for `class_definition`
   - If found, return the class name (first `identifier` child of the class_definition)
   - If not found (module-level code), return None

5. `find_enclosing_function(node: Node, chunks: list) -> Optional[str]`:
   - Walk up parent chain looking for `function_definition`
   - Match by start_line to find the chunk ID
   - Return chunk ID or None
</action>

Add Python relationship extraction functions to `02-worktrees/glma/src/glma/index/relationships.py`:

1. Add import: `from glma.index.resolver import build_import_map, resolve_callee, find_enclosing_class, ImportInfo`

2. `_extract_python_calls(node: Node, source_chunks: list[Chunk], import_map: dict[str, ImportInfo], store: LadybugStore, file_path: str) -> list[Relationship]`:
   - Walk AST looking for `call` nodes
   - For each `call` node:
     - Get callee (first child of `call` node)
     - Get `source_line` = call node start_line (1-indexed)
     - Find enclosing function/method chunk by walking parent chain for `function_definition`, matching start_line to chunk
     - If no enclosing function found (module-level call), skip or use first module-level chunk
     - Call `resolve_callee(callee, import_map, enclosing_class, store, file_path)` â†’ `(target_id, target_name)`
     - Determine confidence:
       - If `target_id` is not empty â†’ DIRECT
       - If callee is `attribute` with object `self` and target_id empty â†’ INFERRED (self.method() where method not found)
       - If callee is plain `identifier` and target found in same file â†’ DIRECT
       - Otherwise â†’ INFERRED
     - Create Relationship(source_id=enclosing_chunk.id, target_id=target_id, target_name=target_name, rel_type=RelType.CALLS, confidence=confidence, source_line=source_line)

3. `_extract_python_imports(node: Node, source_chunks: list[Chunk], import_map: dict[str, ImportInfo], store: LadybugStore, file_path: str) -> list[Relationship]`:
   - For each entry in `import_map`:
     - Convert source_module to file path: "pathlib" â†’ "pathlib.py", "foo.bar" â†’ "foo/bar.py"
     - Query store for file matching that path
     - If file found in store: create Relationship(rel_type=IMPORTS, confidence=DIRECT) from first chunk in current file to first chunk in imported file (or specific imported chunk if imported_name is set)
     - If file not found: create Relationship(rel_type=IMPORTS, confidence=INFERRED) with target_id="" and target_name=source_module
     - source_id = first chunk of current file (or a designated "module" chunk)
     - source_line = line of the import statement in the AST

4. `_extract_python_inheritance(node: Node, source_chunks: list[Chunk], store: LadybugStore, file_path: str) -> list[Relationship]`:
   - Walk AST looking for `class_definition` nodes that have `argument_list` children
   - For each `argument_list` child that contains `identifier` nodes (base class names):
     - Find the class chunk in source_chunks matching the class_definition
     - Look up the base class name:
       a. First check source_chunks for same-file class with that name â†’ DIRECT
       b. Then check store for class chunks with that name in any indexed file â†’ DIRECT
       c. Otherwise â†’ INFERRED with target_name = base class name
     - Create Relationship(rel_type=INHERITS, ...)

5. `extract_python_relationships(filepath: Path, language: Language, repo_root: Path, source_chunks: list[Chunk], store: LadybugStore) -> list[Relationship]`:
   - Get root node
   - Build import map: `import_map = build_import_map(root)`
   - Extract calls: `_extract_python_calls(root, source_chunks, import_map, store, relative_path)`
   - Extract imports: `_extract_python_imports(root, source_chunks, import_map, store, relative_path)`
   - Extract inheritance: `_extract_python_inheritance(root, source_chunks, store, relative_path)`
   - Return combined list

6. Update `extract_relationships()` dispatcher to call `extract_python_relationships` for Language.PYTHON
</action>
