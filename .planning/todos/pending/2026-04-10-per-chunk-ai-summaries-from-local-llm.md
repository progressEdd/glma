## Problem

The `Chunk` model has a `summary` field, but it's never populated. The only AI summarization that exists is file-level (`generate_ai_summary()` in export.py:68), which generates a 1-2 sentence file summary. Individual function/class/method chunks get no AI-generated description — the `chunk.summary` field referenced at export.py:99-100 is always empty, so it contributes nothing to the file-level summary context.

For a code index meant to help agents quickly understand codebases, per-chunk summaries would be high value — a 1-sentence description of what each function does, generated by a local LLM.

## Solution

1. Add a `generate_chunk_summaries()` function (or expand `generate_ai_summary()`) that batch-processes chunks for a file through the local LLM
2. Populate `chunk.summary` during indexing or as an enrichment pass
3. Keep it optional behind a flag (e.g. `--ai-summaries` already exists, just extend its scope)
4. The existing file-level summarizer already reads `chunk.summary` as context (export.py:99-100) — once populated, file summaries automatically improve too
